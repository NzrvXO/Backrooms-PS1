# Backrooms: Echoes - Система уровней и игровой цикл

## Созданные компоненты

### 📊 GameManager
- Отслеживает состояние игры (активированные предохранители, состояние уровня)
- Синглтон - инициализирует уровень при запуске
- Вызывает события: `ActivateFuse()`, `CompleteLevel()`

### 🎮 UIManager
- Показывает количество активированных предохранителей: "FUSES: 0/3"
- Синглтон
- Требует TextMeshPro текстового элемента для UI

### 🏢 Room
- Базовый класс для всех типов комнат
- Типы: Empty, Corridor, Office, Storage, Technical
- Содержит точки спавна для дверей
- Система спавна предохранителей (выше вероятность в Technical комнатах)

### 🔌 Fuse (предохранитель)
- Активируется когда игрок подходит ближе чем `activationDistance`
- При активации: эффект частиц, изменение цвета на зелёный
- Уведомляет GameManager об активации

### 🛗 LiftManager
- Синглтон
- Спавнится в конце уровня
- После активации всех предохранителей:
  - Гасит свет на 2-3 секунды
  - Звучит звук генератора
  - Раздается звонок лифта
  - Открываются двери
- Требует Animator для анимации дверей

### 👁️ Echo (сущность)
- Конечный автомат с состояниями: Idle → Observed → Moving → Disappearing
- Механика наблюдения: если смотреть на Эхо - оно неподвижно, если отвернуться - движется/исчезает
- Случайные позы при спавне (спиной к стене, повернутая голова и т.д.)
- Визуализация: гизмо показывают дистанции наблюдения и исчезновения

### 🏗️ LevelGenerator
- Генерирует процедурный уровень из префабов комнат
- Спавнит лифт в конце
- Спавнит 3 предохранителя в случайных комнатах
- Спавнит 1-3 Эхо в случайных комнатах
- Можно настроить: количество комнат, размер сетки, шанс спавна

### 🌀 AnomalySystem
- Периодически вызывает странные события
- Реализованы:
  - Дубликат комнаты (логика на добавление)
  - Неправильная дверь
  - Манекен (появляется в случайной комнате)

## Как настроить в Unity

### Шаг 1: Создай пустой GameObject "GameManager"
1. Добавь компонент `GameManager`
2. Добавь компонент `UIManager`
3. Добавь компонент `AnomalySystem`
4. В `GameManager` установи ссылку на `LevelGenerator` (пока null, добавим позже)

### Шаг 2: Создай простые префабы комнат
1. Создай 5 кубов (10x3x10) с разными цветами:
   - Empty (белый)
   - Corridor (серый, длинный)
   - Office (бежевый)
   - Storage (коричневый)
   - Technical (стальной)

2. Добавь компонент `Room` к каждому
3. Установи RoomType соответственно
4. Сохрани как префабы в Assets/Prefabs/Rooms/

### Шаг 3: Создай префабы объектов
1. **Fuse**:
   - Куб желтого цвета (0.5x0.5x0.5)
   - Добавь компонент `Fuse`
   - Сохрани как префаб

2. **Echo**:
   - Куб человеческой формы (пока)
   - Добавь компонент `Echo`
   - Добавь Animator (даже пустой)
   - Сохрани как префаб

3. **Lift**:
   - Создай GameObject с дверьми
   - Добавь Collider (IsTrigger)
   - Добавь Light для освещения
   - Добавь AudioSources для звуков
   - Добавь компонент `LiftManager`
   - Добавь Animator для анимации дверей
   - Сохрани как префаб

### Шаг 4: Настрой Player
1. Убедись, что Player имеет тег "Player"
2. В PlayerController установи ссылку на PlayerLook
3. В PlayerLook установи ссылку на Player body

### Шаг 5: Создай LevelGenerator
1. Создай пустой GameObject "LevelGenerator" на сцене
2. Добавь компонент `LevelGenerator`
3. В инспекторе:
   - Установи room prefabs (выбери 5 созданных типов)
   - Установи fusePrefab
   - Установи echoPrefab
   - Установи liftPrefab
4. В GameManager установи ссылку на этот LevelGenerator

### Шаг 6: Создай UI Canvas
1. Создай UI Canvas
2. Добавь TextMeshPro текст для счетчика предохранителей
3. Добавь TextMeshPro текст для сообщений (сделай его неактивным)
4. В UIManager установи ссылки на эти текстовые элементы

## Как это работает

1. При запуске GameManager инициализирует LevelGenerator
2. LevelGenerator генерирует комнаты, спавнит лифт, предохранители и Эхо
3. Игрок исследует уровень и ищет предохранители
4. При подходе к предохранителю - он активируется автоматически
5. После активации всех 3 предохранителей - включается питание лифта
6. Лифт издает звуки и открывает двери
7. Игрок может войти в лифт и "сбежать"
8. Между этим Эхо наблюдают за игроком согласно механике наблюдения
